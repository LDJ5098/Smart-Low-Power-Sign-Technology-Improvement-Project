1단계: 수동 배포에서 자동화의 길로 (Automation)
보안 개념: 사람이 직접 접속할 때 발생하는 '휴먼 에러'와 '자격 증명 노출' 방지.

코드 구현: CI/CD 파이프라인 생성 및 서버 접속 기본 명령어 작성.

실험 기록: "내가 직접 치던 명령어를 봇이 똑같이 수행하는 것을 확인했다."

2단계: 성문을 지키는 장갑차 (SSH Key Auth)
보안 개념: 훔치기 쉬운 비밀번호 대신, 수학적으로 복제가 불가능한 SSH Key 도입.

코드 구현: ssh-keygen으로 열쇠를 생성하고 서버의 authorized_keys에 등록.

실험 기록: "비밀번호 없이 오직 내 열쇠로만 서버에 안전하게 접속하는 '장갑차'를 완성했다."

3단계: 권한의 분리, 마스터와 일꾼 (Separation of Duties)
보안 개념: 나(관리자)와 봇(실행자)의 열쇠를 분리하여 사고 발생 시 피해를 격리함.

코드 구현: 동주 님용과 봇용 SSH Key를 각각 생성하고 개별 관리.

실험 기록: "내 열쇠와 봇 열쇠를 용도에 맞게 완전히 분리하여 관리하기 시작했다." (비번 설정 부분 삭제)

4단계: 흔적 없는 요새 (Stateless & Zero-Trace)
보안 개념: 존재하지 않는 데이터는 해킹당할 수 없다. 로컬의 봇 열쇠를 파기함.

코드 구현: 봇의 개인키를 안전한 **Secret Variables(환경변수)**에 봉인 후 로컬 원본 파일 삭제.

실험 기록: "봇의 열쇠를 안전한 금고에 넣고 내 PC에서 소멸시켜 '완벽한 격리'를 실현했다." (GitLab 명칭 제거)

5단계: 지능형 지연 방어와 타임락 (Intelligent Time-Lock)
보안 개념: 자동화된 공격의 속도를 강제로 늦춰 관리자가 개입할 수 있는 골든타임을 확보함.

코드 구현: 배포 스크립트에 sleep 180(3분 대기) 로직 구현 및 야간 모드 설정.

실험 기록: "배포 시 3분간의 유예 시간을 두어 주인에게 상황을 보고할 여유를 확보했다."

6단계: A·B·C 통합 운영 및 자율 방어 (Fortress Operations)
보안 개념: 상위 권한 접속 시 하위 세션 차단, 배포 전 무결성 검증, 인격화된 로그 기록을 통해 시스템 자율 방어 체계를 완성함.

코드 구현: sshrc 세션 사살, 이미지 해시 비교(무결성), C의 승인/반려 로직, 상황별 커스텀 로그 및 외부 알림 연동.

실험 기록: "관리자가 접속하면 침입자를 쫓아내고, 변조가 감지되면 알람을 울리는 지능형 상호 견제 시스템을 완성했다."

7단계: 최후의 심판, 물리적 단절 (The Last Resort)
보안 개념: 모든 디지털 방어선이 무너진 최악의 상황에서 물리적 존재를 통한 강제 종료 수단을 마련함.

코드 구현: 원격 Root 접속 완전 차단 및 물리 주인(친구)과의 비상 연락망 확립.

실험 기록: "디지털 세계가 무너져도 현실에서 '플러그'를 뽑아 성을 지킬 수 있는 최후의 보루를 확보했다."
